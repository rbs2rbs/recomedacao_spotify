Desenvolvendo uma aplicação de Recomendação do Spotify¶

Resolvi escrever esse texto apenas para tentar explicar melhor como que foi construda a aplicao web que avalia qual msica das 50 mais tocadas no Spotify se parece com uma msica que seja testada pelo usurio. Esse texto est bastante resumido, pelo fato de que a ideia dele apenas comentar como foi feito. Disponibilizei o cdigo apenas da parte que calcula a recomendao.

Em linhas gerais tentarei explicar tudo em 4 partes:

  • Descrio das ferramentas na aplicao
  • Como funciona a API do Spotify
  • Como calculado a semelhana entre as msicas
  • O clculo na prtica

Descrição das ferramentas na aplicação¶

Eu resolvi fazer essa aplicao utilizando o back-end em Python e o front-end utilizando React. Ambas ferramentas me do um certo conforto em trabalhar.

O back-end em Python foi uma escolha quase que essencial, pois pude utilizar vrias bibliotecas muito boas para quem cientista de dados, como Pandas, Numpy, Scikitlearn. Em parceria com essas bibliotecas, que foram responsveis pelos clculos e estimativas, utilizei o framework Django, que ficou responsvel pela criao dos end-points. Neste caso, o back-end havia apenas trs end-points: Um para requisitar a msica que ser comparada, outro para requisitar as 50 musicas mais tocadas atualmente no Spotify, e um ltimo para realizar os clculos e devolver os resultados. No utilizei nada de banco de dados nesta aplicao, por no haver interesse em armazenar nenhuma informao, ao menos at este momento.

Como funciona a API do Spotify¶

O Spotify disponibiliza um servio no qual diversas mtricas do seu contedo so disponibilizados. Com um cadastro e em posse de suas credenciais voc tem acesso. Como suporte utilizei a biblioteca Spotipy do Python. Quando coletei dados de uma msica, diversas informaes so retornados, inclusive caractersticas das msicas hospedadas neste aplicativo. Apenas baseado nas mtricas por eles geradas, consiste essa aplicao. Utilizei uma transformo, de modo a ficar entre zero e um, isso fcilita em aspecto de interpreto e calculos.

Como é calculado a semelhança entre as músicas¶

Para buscar quais msicas parecem com a requisitada pelo usuario fiz uso de um conceito bastante simples que foi nada mais do que a distncia euclidiana entre pontos. De uma forma sucinta explicarei, com um caso de apenas trs variveis, como seria este clculo. Quem j tiver clareza deste conceito pode seguir a prxima seo.

Primeiramente, eu partir de duas variveis muito comuns: Peso e Altura. De forma muito fcil gerei um conjunto de dados com 500 observaes. Assumindo que peso e altura so grandezas que aderem a uma distribuio normal multivariada, com mdias 60kg e 1.66m, respectivamente, e uma matriz de covarincia:

$ \left(\begin{array}{cc} 100 & 0.8\\ 0.8 & 0.0025 \end{array}\right) $

O cálculo na prática¶

Primeiramente um objeto foi criado com funes teis para realizar todo o processo.

Em seguida iniciamos o objeto Funcoes. Nesta etapa feita a requisio de acesso a API. Aqui ficam includas as credenciais.

Utilizando o id da msica escolhida, tive acesso aos dados que queria. Exclui as variveis que no me interessava e transformei as nicas duas variveis que no possuam domnio entre zero e um.

O mesmo processo foi realizado para requisitar as informaes das msicas mais tocadas. Para este caso, o mtodo em questo, primeiramente coleta os id de todas as msicas que pertencem as mais tocadas e em seguida coleta as informaes dos udios em questo. Realizei a mesma transformo feita com a msica solicitada na etapa anterior.

De agora em diante fica fcil. Apenas calculei as distncias euclidianas e verifiquei qual a msica do top 50 que mais se aproxima da solicitada.

    def get_musica(self,music_id):
        sp = self.sp
        saida = {
            'features':sp.audio_features(music_id),
            'track':sp.track(music_id)
        }
        return(saida)

    

    def get_playlist_audio_features(self,username, playlist_id):
        sp = self.sp
        offset = 0
        songs = []
        items = []
        ids = []
        while True:
            content = sp.user_playlist_tracks(username, playlist_id, fields=None, limit=100, offset=offset, market=None)
            songs += content['items']
            if content['next'] is not None:
                offset += 100
            else:
                break

        for i in songs:
            ids.append(i['track']['id'])

        index = 0
        audio_features = []
        while index < len(ids):
            try:
                audio_features += sp.audio_features(ids[index:index + 50])
                index += 50
            except:
                pass
        
        top_null = []
        for i in audio_features:
            if not i is None:
                top_null.append(i)

        return (top_null)
